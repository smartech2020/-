# سامانه هوشمند تحلیل و پیش‌بینی افت تحصیلی دانش‌آموزان
### طرح پژوهشی ویژه مسابقات دانش‌آموزان، مراجع علمی و جشنواره جوان خوارزمی

این سامانه یک پلتفرم جامع دووجهی مجهز به **پشته علمی هوش مصنوعی پیش‌فرض** و **پورتال تحت وب مدرن** است که با هدف **شناسایی زودهنگام دانش‌آموزان در معرض افت تحصیلی** در دوره دوم متوسطه توسعه یافته است. این سیستم فاکتورهای پیچیده و چندبعدی درسی، فردی، بهداشتی و خانوادگی را دریافت کرده و با تحلیل‌های آماری و ریاضیاتی، ریسک افت تحصیلی را برآورد می‌کند.

---

## 📝 ۱. شرح مسئله (Problem Description)

افت تحصیلی یا خروج ناگهانی دانش‌آموز از مسیر مطلوب یادگیری، پدیده‌ای پیچیده، چندعاملی و تدریجی است. پایش سنتی مدارس معمولاً زمانی این افت را تشخیص می‌دهد که نمرات پایانی صادر شده و فرصت طلایی جبران و مداخله هدایت‌گر از دست رفته است. 

عوامل اثرگذار بر وضعیت تحصیلی دانش‌آموزان صرفاً محدود به هوش یا ساعت مطالعه کلاس درس نیست؛ بلکه زنجیره‌ای از متغیرهای درهم‌تنیده بر آن حاکم است:
*   **عوامل فردی و رفتاری:** میزان بهداشت خواب، ساعات استفاده از تلفن همراه و اینترنت، و سطح انگیزه درونی.
*   **عوامل آمادگی آموزشی:** نحوه انجام تکالیف، مشارکت فعال در مباحث کلاس، و تعداد غیبت‌های موجه و غیرموجه.
*   **عوامل حمایتی:** سطح پیگیری و پشتیبانی عاطفی و آموزشی والدین، به همراه برآیند معدل‌های قبلی.

این سامانه با یکپارچه‌سازی ۱۳ متغیر کلیدی رفتارشناسی و تحصیلی در قالب یک سیستم خبره ریاضیاتی (سرور زنده) و مدل‌های یادگیری ماشین آفلاین (نسخه پایتون)، همبستگی‌های غیرخطی میان این فاکتورها را تحلیل کرده و وضعیت هر فرد را پیش از فرارسیدن امتحانات نهایی به صورت هوشمند پیش‌بینی می‌کند.

## 🎯 ۲. اهداف و اهمیت طرح پژوهشی (Goals & Significance)

### اهداف اصلی طرح:
1.  **شناسایی پیشگیرانه (Early Detection):** ردیابی سیگنال‌های پنهان افت تحصیلی (مانند تغییر الگوی خواب یا غیبت‌های مکرر) قبل از تاثیر مستقیم روی نمرات کارنامه.
2.  **ارائه هوش تجویزی و توصیه‌های شخصی‌سازی شده (Actionable Insights):** فراتر رفتن از یک پیش‌بینی ساده و ارائه بسته‌ای از راهکارهای روان‌شناختی و مشاوره‌ای متناسب با ضعف‌های اختصاصی هر دانش‌آموز.
3.  **شفاف‌سازی تصمیمات سیستم (Explainable AI - XAI):** مشخص کردن دقیق اینکه کدام متغیرها بیشترین سهم را در افزایش ریسک افت تحصیلی دانش‌آموز داشته‌اند تا مربیان بتوانند مستقیماً روی نقاط ضعف تمرکز کنند.

### ضرورت و اهمیت کاربردی:
استفاده از این ابزار به مدارس، مشاوران تحصیلی و والدین اجازه می‌دهد تا با حداقل هزینه و به محض بروز اولین نشانه‌های انحراف رفتاری، برنامه‌های حمایتی کلاسی و بهداشتی را آغاز نمایند. این امر منجر به کاهش نرخ مردودی، بهبود انگیزه دانش‌آموزان و نهایتاً ارتقای کیفیت عدالت آموزشی در سطح مدارس می‌گردد.

## 📐 ۳. ساختار و معماری پروژه (Project Directory)

این پروژه از دو ساختار مجزا و بهینه تشکیل شده است تا هم بر روی وب (نسخه پورتال زنده) و هم بر روی رایانه‌های محلی داوران و دانش‌آموزان (نسخه پایتون) به بهترین شکل ممکن اجرا شود:

```text
project/
├── database.json               # پایگاه داده پویا و امن از سوابق ثبت‌شده دانش‌آموزان
├── server.ts                   # وب‌سرور فول‌استک Express پورتال زنده (هسته ریاضیاتی پیش‌بینی و APIها)
├── src/                        # فرانت‌اند مدرن و مصور سامانه (طراحی‌شده با React و Tailwind)
│   ├── main.tsx                # نقطه ورود فرانت‌اند به ساختار برنامه
│   ├── App.tsx                 # رابط کاربری تک‌صفحه‌ای پویا و پیشرفته
│   └── index.css               # فایل تنظیمات استایل RTL و فونت‌های فارسی شبکه‌ای
├── AppPy/                      # ساختار کدهای پورتال محلی آفلاین (مخصوص ویندوز)
│   ├── app.py                  # وب سرور تحت Flask نسخه آفلاین ویندوز
│   ├── train_model.py          # اسکریپت آموزش، مقایسه الگوریتم‌ها و ذخیره فایل مدل هوش مصنوعی
│   └── requirements.txt        # کتابخانه‌های هوش مصنوعی و تحلیل داده پایتونی (scikit-learn, pandas و...)
├── data/
│   └── student_dataset.csv     # ۵۰۰۰ رکورد داده آموزشی شبیه‌سازی‌شده دقیق جهت کالیبراسیون الگوریتم‌ها
├── model/
│   └── academic_risk_model.pkl # فایل فشرده باینری حاوی بهترین مدل بهینه‌شده یادگیری ماشین (.pkl)
└── README.md                   # سند راهنما و تشریح علمی حاضر
```

## 💻 ۴. روش و راهنمای اجرا (Execution & Setup)

### بخش اول: اجرای پورتال زنده تحت وب (محیط ابری / Google AI Studio)
برای ران کردن پورتال پیشرفته React + Node.js در ریشه پروژه، پورت ۳۰۰۰ به صورت خودکار پیکربندی شده است:
1.  سیستم به طور خودکار وابستگی‌های `package.json` را لود می‌کند.
2.  برای تست فرآیند کامپایل و تایید صحت ساختار TypeScript دستور زیر اجرا می‌شود:
    ```bash
    npm run build
    ```
3.  برای اجرای سرور زنده توسعه، از دستور زیر استفاده می‌شود:
    ```bash
    npm run dev
    ```
    سپس برنامه در بستر وب‌سایت لوکال پورت ۳۰۰۰ بالا خواهد آمد.

---

### بخش دوم: راهنمای نصب و راه‌اندازی نسخه محلی پایتون (مخصوص سیستم‌عامل ویندوز)
برای انتقال این پژوهش، تحویل به داوران جشنواره خوارزمی و اجرا در محیط بومی ویندوز، مراحل زیر را گام‌به‌گام در خط فرمان یا CMD سیستم خود بنویسید:

#### ۱. نصب مفسر Python و راه‌اندازی پشته هوش مصنوعی
از نصب بودن پایتون (نسخه ۳.۱۰ یا بالاتر) روی ویندوز اطمینان حاصل کرده و تیک `Add Python to PATH` را در هنگام نصب فعال نمایید. سپس CMD را در ریشه پوشه برنامه باز کرده و دستور زیر را بنویسید:
```bash
pip install -r AppPy/requirements.txt
```
*این دستور تمام ابزارهای حیاتی یادگیری ماشین نظیر Scikit-Learn، Pandas، Numpy، Flask و پلات پایلوت‌های تعاملی را روی ویندوز شما نصب می‌کند.*

#### ۲. آموزش مجدد و ارزیابی مدل‌های ریاضی (ML Training)
برای استخراج همبستگی‌ها و ایجاد بهترین مدل هوش مصنوعی در پوشه `model/` متناسب با ۵۰۰۰ داده تجربی، دستور زیر را اجرا کنید:
```bash
python AppPy/train_model.py
```
*خروجی این دستور، بهترین الگوریتم را از میان **جنگل تصادفی (Random Forest)**، **رگرسیون لجستیک** و **تقویت گرادیان (Gradient Boosting)** انتخاب کرده و شاخص‌های آماری آن را در قالب نمودارهای تحلیلی ذخیره می‌کند.*

#### ۳. اجرای وب‌اپلیکیشن بومی فلاسکی (Flask Server)
در نهایت، برای بالا آوردن رابط کاربری آفلاین روی ویندوز خود، دستور زیر را بنویسید:
```bash
python AppPy/app.py
```
سپس مرورگر خود را باز کرده و وارد آدرس زیر شوید:
👉 **[http://127.0.0.1:5000](http://127.0.0.1:5000)**

## 📊 ۵. خروجی‌های هوشمند سیستم (Expected Outputs)

پس از ثبت اطلاعات یک دانش‌آموز در فرم هوشمند ۱۳ فاکتوری، خروجی‌های زیر به شکل آنی و مصور در دسترس کلاینت قرار خواهند گرفت:

1.  **محاسبه سطح ریسک (Risk Level Evaluation):** تفکیک خروجی به سه سطح بحرانی: **کم‌خطر (سبز)**، **خطر متوسط (زرد)** و **پرخطر / بحرانی (قرمز)** همراه با نمایش درصد دقیق احتمال آسیب تحصیلی.
2.  **درصد اطمینان سیستم (Confidence Score):** محاسبه ریاضی میزان قاطعیت سیستم در پیش‌بینی صادرشده (بین ۷۰٪ تا ۹۹٪) بر اساس سازگاری معیارهای ورودی.
3.  **شفاف‌سازی هوش علمی (Explainable AI - XAI):** نمایش پویای مهم‌ترین فاکتورهای منفی اثرگذار بر افت دانش‌آموز (مثلاً خواب نامناسب یا معدل ناپایدار قبلی) در قالب نمودار میله‌ایِ تفکیک‌شده سهم اثرگذاری.
4.  **نسخه و سیستم توصیه‌گر تربیتی (Actionable Clinical Advisory):** صادر شدن گزاره‌های مشاوره‌ایِ فوری، بهداشتی و درسی متناسب با ضعف‌های اختصاصی ثبت‌شده (به عنوان نمونه، کاهش بهداشت خواب پیامی خاص و برنامه خواب اصولی تولید می‌کند).
5.  **داشبورد آماری مدیریت مدرسه (Managerial Analytics):** نمودارهای پیشرفته توزیع جنسیتی ریسک، هیستوگرام‌های اثر متغیر غیبت کلاسی بر معدل، و بردارهای همبستگی زمان تماشای موبایل و کاهش انگیزه.
